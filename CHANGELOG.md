## 버전 관리 규칙

### 버전 번호 형식: MAJOR.MINOR.PATCH

- **MAJOR**: 호환되지 않는 API 변경
- **MINOR**: 하위 호환되는 기능 추가
- **PATCH**: 하위 호환되는 버그 수정

### 라벨 설명

- ✨ **추가됨**: 새로운 기능 추가
- 🔧 **수정됨**: 기존 기능 수정
- 🐛 **버그 수정**: 버그 수정
- 📚 **문서**: 문서 업데이트
- 🎨 **스타일**: UI/UX 개선
- ⚡ **성능**: 성능 개선
- 🔒 **보안**: 보안 관련 변경
- 🏗 **리팩토링**: 코드 구조 개선
- 🚀 **배포**: 배포 관련 변경
